# Unite Medical 2.0

React + Vite app, FDA-registered, veteran-owned. Built on `react@19`,
`react-router-dom@7`, and a tiny client-side reactive DB (`src/lib/db.js`)
seeded by `src/lib/seed.js`.

```bash
npm install
npm run dev      # dev server on :5173 (or next free port)
npm run build    # production build → ./dist
npm run lint     # eslint
```

## Site rebuild PRD & phase verifier

The active rebuild plan lives at `docs/PRD.md` (5 phases, sourced from the
CTO spec). Items that need real data or external input are tracked in
`docs/ALEX_THINGS.md`. The automated verifier at `scripts/phase_check.py`
is the gating CI step — it fails the build if any forbidden string
survives or any required redirect / route / data file is missing:

```bash
python3 scripts/phase_check.py            # all phases
python3 scripts/phase_check.py --phase 1  # one phase
python3 scripts/phase_check.py --strict   # fail on warnings too
```

The one-shot `scripts/_phase1_replace.py` documents and replays the
codebase-wide string substitutions from spec §3c. It is idempotent.

## Real product catalog

The 87-product Unite Medical catalog (scraped from the live unitemedical.net
Shopify store) is committed at `src/data/realCatalog.js` and rendered through
the in-browser DB. Images live under `public/images/products/{handle}/`.

To regenerate from the upstream CSVs:

```bash
python3 scripts/import_catalog.py
```

Source of truth:

- `/Users/alex-nt-14/Mavera_Main_Bot/unite_medical_catalog/csvs/ALL_PRODUCTS_MASTER.csv` — 87 products
- `/Users/alex-nt-14/Mavera_Main_Bot/unite_medical_catalog/csvs/ALL_VARIANTS.csv` — 280 SKU-level variants
- `/Users/alex-nt-14/Mavera_Main_Bot/unite_medical_catalog/images/{Category}/{handle}/` — original product photos

The importer:

1. Maps Shopify product types onto the site's category vocabulary
   (Orthotics, Diagnostics, PPE, Surgical, Supplements).
2. Copies all product images into `public/images/products/{handle}/`.
3. Emits a single JS module (`src/data/realCatalog.js`) consumed by
   `src/lib/seed.js`, `src/lib/imageMap.js`, and `src/data/index.js`.

After re-running the importer the in-browser DB will reseed automatically
because `SCHEMA_VERSION` is bumped whenever shape changes.

## AI-regenerated hero images (optional)

To replace the original product photos with editorial-style hero images that
match the rest of the site's brand language, run:

```bash
OPENAI_API_KEY=sk-... python3 scripts/generate_catalog_images.py \
    --quality medium     # ~$4.45 across 87 SKUs
# or
OPENAI_API_KEY=sk-... python3 scripts/generate_catalog_images.py \
    --quality low        # ~$0.50 across 87 SKUs (debug)
# or
OPENAI_API_KEY=sk-... python3 scripts/generate_catalog_images.py \
    --quality high       # ~$18.40 across 87 SKUs (final)
```

It uses OpenAI's `images.edit` endpoint with the original Shopify hero photo
as a reference so the generated image preserves shape, color, label text, and
branding. Output lands in `public/images/products-ai/{handle}.png`. To make
those the live hero images, swap the path in `src/lib/imageMap.js` (or copy
them over the originals before committing).

## Marketing imagery

The editorial hero / segment / blog imagery (`public/images/generated/*.png`)

and is unrelated to the product catalog above.
