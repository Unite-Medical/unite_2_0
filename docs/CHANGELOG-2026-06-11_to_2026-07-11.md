## 8. Vendor onboarding pipeline (Jul 7)

Damon's Jul 6 quoting-engine briefing, built end to end — all eight §8 checks:

- **53-column v2 template ingestion** — parser covers every field (product, pricing with price breaks, classification, FDA compliance, carton/pallet logistics) with fuzzy header matching + AI fallback.
- **HTS duty bug fixed** — blank HTS no longer silently defaults to a generic textile code; the manufacturer's HS code becomes a hint and the line is flagged HTS-pending until desk confirmation.
- **Section 301 pre-filter** — Chapter 99 lookup applies List 1–4A surcharges (25%/7.5%) on China-origin goods automatically; combined MFN + 301 duty math.
- **Flexport classification** — new API call (confirm duties) + one-click export of the classification file in Flexport's exact column layout.
- **Real freight math** — CBM/weight from actual carton dimensions; origin port resolved from the vendor's sheet (was hardcoded).
- **SKU-match engine** — every uploaded line checked against the catalog (exact + fuzzy); in-stock lines get a buy-now badge on the quote.
- **Offer variants** — every line carries side-by-side **Unite Ready** (stock or import) and **Unite Custom** (private label, tooling amortized, sample cost/lead shown).
- **No-quote feedback loop** — unmatched/unparseable/dropped lines captured as demand signals with reason codes and a desk resolution queue (`quote_misses`).
- **Template v4.0** — downloadable vendor template regenerated to the 53-column spec with color-coded required/requested headers.
- **Test infrastructure introduced** — `node --test` suite (32 tests at this point) + `npm test`.

## 10. GUDID / UDI compliance-as-a-service (Jul 10)

Damon's Phase 1 spec, built same-day:

- **GS1 prefix registry** — all 5 prefixes (Unite ×4, Medava ×1) with medical flags; device DIs can only be issued from medical-flagged prefixes; new Unite assignments draw from the fresh `0850089282` prefix.
- **DI assignment engine** — sequential GTIN-13 issuance with valid GS1 check digits; per-prefix used-vs-remaining tracking; near-cap alert at ≤20 remaining.
- **Labeler models A/B/C** — A (Unite Ready: our DI, we submit), B (customer has GS1: their DI, they submit), C (Unite as labeler of record: **hard-blocked until the customer signs all four acknowledgment terms** — GUDID accuracy, complaints, MDR reporting, recalls). The paid-service moat, enforced in code.
- **Class 1 / Class 2 intake** — validation matching Damon's workbook (21 core fields; Class 2 requires a real K-number, Class 1 takes "Exempt"); in-app intake editor with live per-field validation; downloadable intake workbook generated from the same field list so template and validator cannot drift.
- **Both label paths** — generate from field values (Model C labels automatically carry "Distributed by Unite Medical"; PI placeholders from flags) or compliance-check uploaded artwork against the required-elements list.
- **Post-quote / pre-production gate** — opens automatically on order commit for import/private-label lines without a GTIN; never blocks a quote or acceptance; desk task created; PI values captured at production, flags at intake.
- **`/admin/udi` desk page** — prefix capacity board, gate queue with inline model/class intake, Model C acknowledgment recorder, label actions, portal-submission tracking.
- **Inbox rerouting** per Damon: `billing@` → `accounting@`; `privacy@`/`surplus@` → `support@`.
